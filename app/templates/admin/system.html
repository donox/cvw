Pull the latest code from GitHub. Uvicorn reloads automatically. Use this after pushing new code.

Restart Server

Restart the application container. Required to apply a staged database restore. The server will be unavailable for ~15 seconds.

When is a terminal deploy required instead of Deploy Update?

Deploy Update (this page) does a git pull and lets Uvicorn reload the Python files. That covers most code changes.

A terminal deploy is required when:

  • New Python packages were added — git pull alone won't install them; the Docker image must be rebuilt with docker compose up -d --build.
  • The Dockerfile or docker-compose.yml changed — same reason; a rebuild is needed.
  • A migration failed or was stamped manually — you may need to run python -m alembic upgrade head or a patch script inside the container.

The deploy output above will say "If new packages were added, a manual docker compose up -d --build is required" when it detects changed files — but when in doubt, a full terminal deploy is always safe.

What is a staged database restore?

When you click Restore on the Backups page, the backup file is copied to a holding location but the live database is not touched yet. This is the "staged" state — nothing is overwritten until the server restarts.

On restart, the app detects the staged file, swaps it in as the live database, and starts normally with the restored data. The yellow "Restore staged" banner at the top of this page appears when a restore is waiting.

The two-step design means you can cancel a restore simply by restarting without confirming — though once the restart begins the swap happens automatically. Always take a fresh manual backup before restoring, so you can recover the state you had just before.
